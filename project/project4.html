QUICK ACCESS INSURANCE WEBSITE UI

Prototype/Front-end · UI/UX Design

Project Background

A desktop-friendly prototype for a fictional insurance platform focused on quick access to core features. The goal was to simplify navigation and build trust through clear, modern design.

Problem Statement

Insurance sites are often cluttered and hard to navigate, creating friction for users seeking quick registration or policy info. This design aims to streamline the process with a clean, intuitive interface.

My Role

UI Designer — created the layout, user flow, and high-fidelity prototype using Figma.

Tools Used

Figma

Prototype

Accessibility Considerations

Used clear step-by-step navigation and strong color contrast for readability. Required fields were marked visually and paired with concise error messages for clarity.

UX Considerations

Designed a guided multi-step flow to reduce cognitive load. Incorporated inline validation, microcopy for hints, and emphasized essential fields without overwhelming users.

Next Steps

Future improvements include adding policy lookup, document uploads, and live chat support. Usability testing with new users will help validate the guided flow approach.

Reflection

This project reinforced how step-by-step design reduces friction in complex domains like insurance. Simplifying forms and guiding users can transform a tedious process into an easier experience.
