CPI Website Revamped

Client Project · WordPress Developer / UX Implementation

Project Background

This internal tool was developed in collaboration with the HR department to streamline recruitment. Previously, hiring was managed through platforms like LinkedIn and other external job boards. This project marked a shift toward hosting job listings and applications directly on the company website, offering more control, better branding, and an improved candidate experience.

Problem Statement

The existing recruitment process relied heavily on third-party platforms and manual processing, making it inefficient and disconnected. Application forms lacked structure, HR had no centralized database, and candidates often didn’t receive timely updates. The goal was to create a dedicated, user-friendly portal that automated essential steps and brought the hiring process in-house.

My Role

WordPress Developer — responsible for plugin development, custom application forms, email integration, and close collaboration with HR to implement features tailored to their needs.

Key Contributions

  • Developed custom WordPress plugins using PHP, HTML/CSS, and JavaScript.
  • Created structured application forms with logic, validation, and automated email notifications.
  • Integrated HR feedback to ensure usability for both staff and applicants.
  • Streamlined the workflow with backend tools that made application tracking easier for HR.
  • Provided post-launch support and updates to refine the system based on live usage.

Tools Used

WordPress, PHP, JavaScript, HTML/CSS, SMTP Email Integration

UX Considerations

Form fields were grouped logically, with real-time validation and placeholder hints to prevent errors. Plain language labels, intuitive navigation, and success/error states improved clarity for applicants. I also worked closely with HR to ensure that the interface required minimal training to use.

Accessibility Considerations

All form fields included semantic labels for screen reader compatibility. Contrast ratios were tested to ensure readability for visually impaired users. Required fields were marked clearly, and error states used color and text to ensure clarity.

Next Steps

Future improvements could include an applicant dashboard, file upload status indicators, analytics for HR, and a built-in scheduler for interviews. These additions would continue to automate and enhance the recruitment workflow.

Reflection

This project helped me deepen my WordPress development skills and highlighted the value of cross-functional collaboration. Transitioning HR from third-party platforms to an internal tool was a big shift, and seeing them adapt to a more streamlined and branded hiring experience was especially rewarding.
